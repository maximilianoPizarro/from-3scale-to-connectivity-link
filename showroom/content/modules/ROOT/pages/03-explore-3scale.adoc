= Explore the 3scale Environment (Migration Source)

In this module you will inspect the two applications that are deployed with *Red Hat 3scale API Management*. These are the *migration source* — the same applications also exist in Connectivity Link namespaces for comparison.

== Part 1 — Neuralbank on 3scale (OIDC)

The `neuralbank-3scale` namespace contains the Neuralbank banking API protected by a 3scale *Product* with OIDC authentication via Keycloak.

=== 1.1 Inspect the running pods

[source,bash]
----
oc get pods -n neuralbank-3scale
----

You should see the backend, frontend, and database pods:

[cols="1,2",options="header"]
|===
| Pod | Role
| `neuralbank-backend-*` | Quarkus REST API
| `neuralbank-frontend-*` | Web SPA
| `neuralbank-db-*` | PostgreSQL database
|===

=== 1.2 Inspect the 3scale Product

The 3scale Product defines the API's public endpoint, authentication method, and rate limits:

[source,bash]
----
oc get product -n 3scale-system | grep neuralbank
----

Inspect the Product configuration:

[source,bash]
----
oc get product neuralbank-oidc-product -n 3scale-system -o yaml
----

Key fields to note:

* `spec.deployment.apicastSelfManaged` — APIcast handles traffic
* `spec.authentication.oidc.issuerEndpoint` — Points to the Keycloak realm
* `spec.mappingRules` — Defines which URL patterns map to metrics
* `spec.applicationPlans` — Rate limits per plan (basic/premium)

=== 1.3 Inspect the 3scale Backend

[source,bash]
----
oc get backend -n 3scale-system | grep neuralbank
----

The Backend resource defines the *private base URL* that APIcast forwards traffic to (the internal Kubernetes Service).

=== 1.4 Check the ActiveDoc (OpenAPI)

[source,bash]
----
oc get activedoc -n 3scale-system | grep neuralbank
----

The ActiveDoc attaches the OpenAPI spec to the Product, making it available in the 3scale Developer Portal.

== Part 2 — NFL Wallet on 3scale (API Key)

The `nfl-wallet-3scale` namespace contains the NFL Wallet API protected by a 3scale *Product* with API Key (`user_key`) authentication.

=== 2.1 Inspect the running pods

[source,bash]
----
oc get pods -n nfl-wallet-3scale
----

=== 2.2 Inspect the 3scale Product

[source,bash]
----
oc get product -n 3scale-system | grep nfl-wallet
----

[source,bash]
----
oc get product nfl-wallet-apikey-product -n 3scale-system -o yaml
----

Key differences from the Neuralbank Product:

* `spec.authentication.userkey` — Uses `user_key` query parameter instead of OIDC
* Applications hold a static `user_key` value
* APIcast validates the key against the 3scale internal database

=== 2.3 Inspect the Backend and ActiveDoc

[source,bash]
----
oc get backend -n 3scale-system | grep nfl-wallet
oc get activedoc -n 3scale-system | grep nfl-wallet
----

== 3scale resource summary

[cols="2,3,3",options="header"]
|===

|===
